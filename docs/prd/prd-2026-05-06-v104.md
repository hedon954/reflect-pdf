# LumenPDF — 产品需求文档 (PRD)

**版本**: v1.0.4 · **日期**: 2026-05-06

---

## 本次迭代概述 (v1.0.4)

v1.0.4 是一次**翻译体验**专项优化，核心目标是把"等翻译"的感知延迟从「秒级肉眼可见的卡顿」降到「几乎无感」。本版本不引入新业务功能，但显著改善每一次翻译的可见行为：

1. **翻译切换为流式输出**——边生成边显示，不再等全部 7 个字段都生成完才出现
2. **气泡渐进渲染**——已生成字段立即可见，配状态指示器
3. **HTTP 超时与兜底降级**——LLM 网关挂死时不再无限转圈

---

## 新增 / 变更功能需求

### O12 — 翻译流式输出（首字时间优化）

**问题**：
当前翻译流程是「同步阻塞」：UI 调用 `translate(...)` 后必须等 LLM 完整生成所有字段（单词模式 7 个字段，含 2–3 段长文本）才返回。一次典型请求耗时 3–8 秒，期间气泡只显示一个 spinner，用户没有任何"在前进"的反馈。

**根因（量化）**：

| 因素 | 占比 |
| --- | --- |
| 非流式：等所有 token 输出完才返回 | 60–80% |
| 输出 tokens 多（7 字段 + 2–3 段长文本） | 10–15% |
| 网络 RTT / TTFT | 5–10% |

**需求**：

- 翻译请求改为**流式（Server-Sent Events）**消费
- LLM 生成的每一个字段（如 `word` → `phonetic` → `context_translation` → ...）一旦完整到达，立即可见
- 单词模式（`translate`）和句子模式（`translateSentence`）都需支持流式
- 缓存命中场景保持原有行为（一次性立即显示）
- LLM 失败回落到 MyMemory 兜底时，保持单次显示行为（兜底 API 不支持流式）
- 不破坏现有 `translate` / `translateSentence` 同步 API（保留供调用方按需使用）

**预期效果**：

| 指标 | v1.0.3 | v1.0.4 |
| --- | --- | --- |
| 首字时间（看到任何反馈） | 3–8 秒 | **0.5–2 秒**（≈ TTFT） |
| 看到完整内容 | 3–8 秒 | 与 v1.0.3 相当（略快） |
| 视觉状态 | 大 spinner，全空白 | 字段一行行出现 |

---

### O13 — 翻译气泡渐进渲染

**问题**：
即使后端是流式的，如果前端仍按"全部就绪才渲染"，用户看不到差异。气泡的加载态与内容态需要重新设计。

**需求**：

- 只要至少有一个字段非空，气泡立即切换到内容渲染模式
- 顶部展示一个细长的 **「正在生成…」** 状态条（含 spinner），明确表达"还在写"
- 流式期间**不显示「保存到单词本 / 笔记」按钮**——避免用户提前点击保存空数据
- 流式完成后状态条消失，保存按钮出现，行为同 v1.0.3
- 失败完全失败（LLM + 兜底都失败）依旧走原有的「翻译失败」展示

**交互细节**：

- 字段更新时无闪烁（依赖 SwiftUI 对 `TranslationResult` 的 `Equatable` diff）
- 用户在流式中途**关闭气泡**或**重新选词**：旧请求的后续回调被丢弃，不会污染新气泡

---

### O14 — HTTP 超时与连接复用

**问题**：

- 当前 `reqwest::Client::new()` 默认**无超时**：LLM 网关挂死或 504 时，UI 一直 spinner
- LLM 翻译器和兜底翻译器各自维护一个 `OnceLock<Client>`，重复代码

**需求**：

- 统一一个共享 HTTP 客户端，被所有翻译器复用（连接池 + TLS 会话缓存共享）
- 强制超时：
  - **connect_timeout = 3 秒**：3 秒内连不上就放弃，触发兜底
  - **总 timeout = 30 秒**：覆盖整个流式生命周期（一次完整翻译的合理上限）
  - **pool_idle_timeout = 60 秒**：避免长时间不用时连接池里僵尸连接
- 超时触发时返回明确错误，由 domain service 接住后降级到 MyMemory

**预期效果**：

- 网关挂死场景下，30 秒后必能看到兜底译文或失败提示，不再出现"长时间转圈、无人回应"

## 界面变更

### 翻译气泡（流式期间）

```
┌─────────────────────────────────┐
│ investigate                     │
│ [/ɪnˌvɛstɪˈɡeɪt/]               │
├─────────────────────────────────┤
│ ⟳ 正在生成…                     │
│                                 │
│ 语境翻译                        │
│ 调查、研究                      │
│                                 │
│ (其它字段陆续填充中…)            │
└─────────────────────────────────┘
```

### 翻译气泡（完成后）

与 v1.0.3 相同：状态条消失，保存按钮显示。

## 兼容性与回归

- **API 兼容**：`translate` / `translateSentence` 同步接口保留并继续可用，新接口为 `translateStreaming` / `translateSentenceStreaming`
- **缓存行为**：流式成功仍写本地 SQLite 缓存（与 v1.0.3 一致）；兜底 / 失败结果不写缓存
- **数据库**：本次迭代**不涉及 schema 变更**

---

## 验收标准

| 用例 | 期望行为 |
| --- | --- |
| 单词翻译（首次） | 首字 ≤ 2s 出现，字段陆续填充 |
| 单词翻译（缓存命中） | 立即显示完整结果（与 v1.0.3 一致） |
| 句子翻译 | 译文边生成边显示 |
| LLM 网关挂死 | ≤ 30s 自动降级到 MyMemory，气泡显示「基础翻译」标识 |
| 流式中途关闭气泡 | 静默，无残留更新写入新选词 |
| 流式中途选新词 | 新气泡正确显示新词翻译，不被旧请求干扰 |
