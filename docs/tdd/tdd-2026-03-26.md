## 1. 单词卡片弹窗高度自适应

**问题**：`TranslationBubble` 内容区 `ScrollView` 固定 `maxHeight: 260`，内容少时卡片显得空洞，内容多时又立即裁切。

**方案**：将上限从 260pt 提升至 520pt，让 SwiftUI 的 `ScrollView` 在内容不足时自然收缩（`ScrollView` 的高度由内容决定，`frame(maxHeight:)` 只设置上限）。

```swift
// TranslationBubble.swift — content body
ScrollView { ... }
    .frame(maxHeight: 520)   // 原来 260，改为 520
```

效果：

- 短内容（仅语境翻译）→ 卡片自动收小，无多余空白。
- 长内容（多节 + 原文语境 + LLM 错误）→ 最多展开到 520pt，超出后可滚动。

## 2. 首次启动自动弹出 LLM 配置弹窗

### 2a. 清空默认配置值

**问题**：`SettingsView` 的 `@AppStorage` 预置了 `https://api.openai.com/v1` 和 `gpt-4o-mini`，导致新用户首次打开时 UserDefaults 中已有非空值，跳过了配置引导；同时 Rust 后端会用这些默认值尝试调用接口并报错。

**方案**：默认值改为空字符串，用 TextField placeholder 提示用户应填写什么格式。

```swift
// SettingsView.swift
@AppStorage("llm_base_url") private var baseURL = ""
@AppStorage("llm_model")    private var model    = ""

TextField("例：https://api.openai.com/v1", text: $baseURL)
TextField("例：gpt-4o-mini",               text: $model)
```

> API Key 始终从 Keychain 读写，不受 AppStorage 影响，无需改动。

### 2b. 启动时检测并弹出配置 Sheet

**触发条件**：`baseURL`（AppStorage）或 `llm_api_key`（Keychain）任一为空时，在 `ContentView.onAppear` 中将 `showSetupSheet` 置为 `true`。

```swift
// ContentView.swift
@State private var showSetupSheet = false
@AppStorage("llm_base_url") private var baseURL = ""

.onAppear {
    let storedKey = KeychainService.load(key: "llm_api_key") ?? ""
    if baseURL.isEmpty || storedKey.isEmpty {
        showSetupSheet = true
    }
}
.sheet(isPresented: $showSetupSheet) {
    SettingsView().environmentObject(appState)
}
```

**设计决策**：

- 使用 `.sheet` 而非系统设置窗口（`SettingsLink`），因为 Sheet 可在主窗口出现前弹出、关闭后自动回到阅读界面，体验更流畅。
- 检测逻辑仅在 `onAppear` 触发一次，用户关闭 Sheet 后不会重复弹出（即使配置仍不完整）；下次重启再次检测。
- 已有配置的用户（AppStorage 有值 + Keychain 有 Key）不受影响，直接进入主界面。
