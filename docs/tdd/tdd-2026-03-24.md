## 1. 新功能

### 1.1 工具栏文件名与页码（`ContentView.swift`）

**原来**：工具栏中央仅有一个 Picker 用于切换 Tab。  
**现在**：

```
[ PDF 阅读 | 单词本 ]  |  Designing Data-Intensive Applications.pdf  372 / 673
```

实现要点：

- `ToolbarItem(placement: .principal)` 内改为 `HStack`，依次排列：分段 `Picker`（宽 200）→ `Divider` → 文件名 → 页码。
- 文件名使用 `.fixedSize()` 允许自然宽度展开，用 `.lineLimit(1)` + `.truncationMode(.middle)` 优雅处理过长名称；不再使用 `frame(maxWidth: 200)` 硬截断。
- 页码 `Text("\(appState.currentPageIndex + 1) / \(appState.totalPages)")` 使用 `.monospacedDigit()` 防止数字宽度跳动，设置 `.animation(.none)` 避免翻页时数字抖动。

### 1.2 多行选区精确标注（`PDFReaderView.swift`）

**问题**：跨行选词时，以单个大矩形框覆盖两行之间的空白，导致高亮/划线不精确。

**实现**：

```swift
// selectionChanged 中
let rawLines = selection.selectionsByLine()
let lineSelections = rawLines.isEmpty ? [selection] : rawLines
let lineRects = lineSelections.compactMap { s -> CGRect? in
    let r = s.bounds(for: currentPage)
    return r.isEmpty ? nil : r
}
// 多个矩形用 "|" 拼接："{x,y},{w,h}|{x,y},{w,h}"
let boundsStr = lineRects.isEmpty
    ? NSStringFromRect(overallBounds)
    : lineRects.map { NSStringFromRect($0) }.joined(separator: "|")
```

- `SelectionInfo` 新增 `boundsStr: String` 字段，贯穿 `onTextSelected` → `translationRequest` → 标注创建 → SQLite 存储全链路。
- `parseAnnotationRects(_:)` 静态方法：反序列化管道分隔字符串为 `[CGRect]`，向后兼容单矩形（无 `|`）的旧数据。
- `addVocabAnnotation`、`addFreeAnnotation` 均对每个 `lineRect` 分别创建一个 `PDFAnnotation`，彻底消除行间空白缝隙。

### 1.4 LLM 配置热更新（`api.rs` + `BridgeService.swift` + `SettingsView.swift`）

**问题**：修改 LLM 设置后需要重启 App 才生效。

**Rust 侧**：

```rust
// 旧：OnceLock，只能写一次
static LLM_CONFIG: OnceLock<LlmConfig> = OnceLock::new();

// 新：RwLock，运行时可任意更新
static LLM_CONFIG: RwLock<Option<LlmConfig>> = RwLock::new(None);
```

- 新增 `#[uniffi::export] pub fn update_llm_config(config: AppConfig) -> Result<(), LumenError>`，只更新配置，不动 DB 连接池。
- `initialize` 改为幂等：POOL 已存在时跳过创建，但始终更新 LLM 配置；移除 `OnceLock::set` 的冲突报错。
- `llm_config()` 改为返回 clone（`LlmConfig` 全字段为 `String`，代价极低），确保每次翻译都拿到最新配置。

**Swift 侧**：

```swift
// BridgeService.updateConfig — 旧（需重新初始化）
isInitialized = false
initializeIfNeeded()

// 新（直接热更）
try? _updateLlmConfig(AppConfig(...))
```

`SettingsView.saveSettings` 调用 `BridgeService.shared.updateConfig`，保存即生效，无需重启。

### 1.5 LLM Prompt 增强：OCR 拼写纠正（`llm_translator.rs`）

PDF 文本提取常见问题：行尾连字符 `investi-\ngating`、OCR 识别偏差。

新 Prompt 明确指示模型：

```
IMPORTANT: The selected text may contain OCR errors, line-break hyphens (e.g. "investi-\ngating"), or extra whitespace due to PDF extraction. In the "word" field, output the correctly spelled, properly joined word.
```

JSON 模板中 `"word"` 字段说明改为：  
`"correctly spelled word (fix any hyphenation, OCR errors, or typos from PDF extraction)"`

## 2. Bug 修复

### 2.1 阅读位置最小化后丢失

**根因**：`PDFKitView` 的 `savedPage` / `savedScrollOffset` 来自 `PdfDocument`（`document.lastPage`）。会话内每次滚动只写 SQLite 与 `AppState.currentPageIndex`，**`selectedDocument` 快照从不更新**。最小化后若视图重建，`updateNSView` 用陈旧的 `document.lastPage` 作为恢复基准，回到本次打开 App 时的页码。

**修复**：

```swift
// AppState 新增
@Published var currentScrollOffset: Double = 0

// selectedDocument.didSet 中初始化
currentScrollOffset = doc.lastScrollOffset

// saveReadingPosition 中同步
currentScrollOffset = scrollOffset

// PDFReaderView 改为从 AppState 读取
savedPage: appState.currentPageIndex,
savedScrollOffset: appState.currentScrollOffset,
```

`AppState` 成为会话内阅读状态的唯一权威来源，不再依赖过期的 `PdfDocument` 快照。

---

### 2.2 document 赋值触发 page 0 导致 TOC 回滚

**根因**：`pdfView.document = doc` 会**同步**触发 `PDFViewPageChanged`，此时当前页为 0。原来的 `pageChanged` 立即调用 `saveReadingPosition(page: 0)`，把 SQLite 和 `AppState.currentPageIndex` 写成 0，TOC 高亮回到第一章。

**修复**：在 `pdfView.document = doc` **之前**设置 `pendingRestoreTargetPage`：

```swift
// updateNSView 中（document = 之前）
context.coordinator.pendingRestoreTargetPage = savedPage
context.coordinator.lastScrollOffset = savedScrollOffset
context.coordinator.schedulePendingRestoreTimeout()  // 2 s 超时兜底
pdfView.document = doc
```

```swift
// pageChanged 中
if let target = pendingRestoreTargetPage {
    if pageIndex != target { return }         // 忽略中间的 page-0 等
    pendingRestoreTargetPage = nil
    lastKnownPageIndex = pageIndex
    parent.onPageChange(pageIndex, lastScrollOffset)  // 用 DB 值，非测量值
    return
}
```

- `outlineNavigate` / `jumpToPage` 用户主动跳转时清除 `pendingRestoreTargetPage`，避免恢复期间锁住目录导航。

---

### 2.3 连续滚动模式下恢复位置不准

**根因**：单页 / 连续滚动模式下，`pdfView.go(to: page)` 只跳到页面顶部，无法恢复**页内**纵向位置。

**修复**：新增 `applyNormalizedScrollOffset`：

```swift
static func applyNormalizedScrollOffset(_ normalized: Double, to pdfView: PDFView) {
    guard let sv = pdfView.enclosingScrollView, let dv = sv.documentView else { return }
    let h = dv.bounds.height
    guard h > 0 else { return }
    let y = CGFloat(max(0, min(1, normalized))) * h
    let visibleH = sv.documentVisibleRect.height
    let maxY = max(0, h - visibleH)
    sv.contentView.scroll(to: NSPoint(x: 0, y: min(y, maxY)))
}
```

在 `go(to:)` 之后延迟 0.25s（等待 layout 完成）再应用：

```swift
DispatchQueue.main.asyncAfter(deadline: .now() + 0.25) {
    Coordinator.applyNormalizedScrollOffset(savedScroll, to: pdfView)
}
```

窗口还原后同样恢复纵向偏移：

```swift
@objc func windowDidDeminiaturize(...) {
    pdfView.go(to: page)
    DispatchQueue.main.asyncAfter(deadline: .now() + 0.2) {
        Self.applyNormalizedScrollOffset(offset, to: pdfView)
    }
    NotificationCenter.default.post(name: .windowDidDeminiaturize, object: nil)
}
```

---

### 2.4 窗口通知注册失效

**根因**：窗口通知（`willMiniaturizeNotification` / `didDeminiaturizeNotification`）被放在 `updateNSView` 内部的异步闭包中注册，但 `updateNSView` 在文档已加载时 `guard … else { return }` 直接退出，异步块**永远不执行**。

**修复**：移至 **`makeNSView`** 中一次性注册（`object: nil` 监听所有窗口），在处理方法内通过 `pdfView.window === notification.object` 过滤：

```swift
// makeNSView
nc.addObserver(coordinator, selector: #selector(windowWillMiniaturize(_:)),
               name: NSWindow.willMiniaturizeNotification, object: nil)
nc.addObserver(coordinator, selector: #selector(windowDidDeminiaturize(_:)),
               name: NSWindow.didDeminiaturizeNotification, object: nil)

// 处理方法内过滤
guard let notifWindow = notification.object as? NSWindow,
      pdfView.window === notifWindow else { return }
```

---

### 2.5 TOC `scrollTo` 对屏幕外条目无效

**根因**：`ScrollView` + `LazyVStack` 仅渲染可视区域内的条目；目标条目未被 layout，`ScrollViewReader.scrollTo(id:)` 静默失败。

**修复**：改为 `VStack`（一次性渲染全部大纲条目）：

```swift
// 旧
LazyVStack(alignment: .leading, spacing: 0) { ... }

// 新
VStack(alignment: .leading, spacing: 0) { ... }  // scrollTo 可定位任意条目
```

---

### 2.6 TOC 不跟随阅读位置（`onAppear` + 窗口通知）

新增两处补充滚动逻辑：

```swift
// 启动时：延迟 0.5s 等 PDFView 恢复页码
.onAppear {
    DispatchQueue.main.asyncAfter(deadline: .now() + 0.5) {
        scrollProxy.scrollTo(activePageIndex, anchor: .center)
    }
}

// 最小化还原后：onChange 不触发（页码不变），用自定义通知补救
.onReceive(NotificationCenter.default.publisher(for: .windowDidDeminiaturize)) { _ in
    DispatchQueue.main.asyncAfter(deadline: .now() + 0.3) {
        scrollProxy.scrollTo(activePageIndex, anchor: .center)
    }
}
```

### 2.7 阅读位置保存不及时（Cmd+S / 退出前）

新增三条保存时机：

| 时机                  | 实现                                                                                                                    |
| --------------------- | ----------------------------------------------------------------------------------------------------------------------- |
| `Cmd+S` 手动触发      | ZStack 内隐藏 `Button`（`.opacity(0)` + `.keyboardShortcut("s", modifiers: .command)`）发 `saveReadingPositionNow` 通知 |
| 实时滚动（防抖 0.5s） | `NSScrollView.didLiveScrollNotification` → `scrollDebounce` timer；`pendingRestore` 期间跳过                            |
| App 退出前            | `NSApplication.willTerminateNotification` → `appWillTerminate` 同步调用 `BridgeService.saveReadingPosition`             |
| 最小化前              | `NSWindow.willMiniaturizeNotification` → `windowWillMiniaturize` 同步保存                                               |

---

### 2.8 Security-Scoped Bookmark 路径不一致

**根因**：通过安全作用域书签打开的文件，`PDFDocument.documentURL?.path` 可能与原始 `filePath` 不同（解析后路径 vs 用户选择路径），导致所有以 `documentURL?.path == filePath` 为条件的通知过滤失效。

**修复**：`Coordinator` 新增 `var currentFilePath: String`，在 `updateNSView` 中赋值为传入的 `filePath`；所有通知处理方法均改为 `filePath == currentFilePath`。

---

### 2.9 `saveReadingPosition` 触发 `refreshLibrary` 引发状态抖动

每次滚动保存都调用 `refreshLibrary()`（数据库查询 + `@Published` 刷新），与视图恢复逻辑产生竞争，引发 TOC 重绘、位置回滚等副作用。

**修复**：从 `saveReadingPosition` 中移除 `refreshLibrary()`；文库更新只在打开/删除文件时触发。

## 3. 状态管理变更（`AppState.swift`）

```swift
// 新增
@Published var currentScrollOffset: Double = 0
@Published var totalPages: Int = 0

// selectedDocument.didSet 中新增初始化
currentPageIndex = Int(doc.lastPage)
currentScrollOffset = doc.lastScrollOffset
totalPages = Int(doc.totalPages)

// saveReadingPosition 中同步
currentScrollOffset = scrollOffset
```

`AppState` 现在是会话内阅读状态（页码 + 纵向偏移 + 总页数）的单一来源，`PdfDocument` 仅作启动时的初始快照。

## 4. 数据流对比

### 阅读位置保存路径（修复后）

```
用户滚动
  │
  ├─ (0.5s 防抖) NSScrollView.didLiveScrollNotification
  │     → Coordinator.didLiveScroll
  │           → AppState.saveReadingPosition(page, offset)
  │                 → SQLite + AppState.currentPageIndex/currentScrollOffset
  │
  ├─ Cmd+S
  │     → Coordinator.savePositionNow → AppState.saveReadingPosition
  │
  ├─ 最小化前  willMiniaturizeNotification
  │     → Coordinator.windowWillMiniaturize → BridgeService.saveReadingPosition
  │
  └─ App 退出  willTerminateNotification
        → Coordinator.appWillTerminate → BridgeService.saveReadingPosition
```

### 阅读位置恢复路径（修复后）

```
document 赋值前：pendingRestoreTargetPage = savedPage（来自 AppState）
document = doc
  └─ PDFViewPageChanged (page=0) → 被 pendingRestore 拦截，return
  └─ PDFViewPageChanged (page=savedPage) → 清除 pending，onPageChange(page, DB offset)

go(to: savedPage) + 0.25s 延迟 applyNormalizedScrollOffset(savedScroll)
```
