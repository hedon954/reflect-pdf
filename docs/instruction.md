我要做一个用于阅读 PDF 的小工具，它的核心是参考 Mac 的预览工具（界面简洁，性能好，流畅），但是不同的是（也是我要解决的核心痛点），我希望它可以划词翻译，不一样的是什么呢？第一，它这个翻译，跟普通的翻译不一样，是可以基于上下文的翻译（比如自动识别到这个词所在的句子，将整个句子一起发给 LLM 进行翻译，不止是告诉我这个词是什么意思，而且是告诉我为什么是这个意思。第二，翻译后，后一个“保存”按钮，这样我可以快速将这个单词、所在的句子、和 LLM 的解释，保存到单词本中，同时，在 PDF 中，对这个词进行高亮。这样我就可以很方便进行复习了，而且，后续重复选择这个位置的这个词的话，就不需要重复调用 LLM，可以省钱，也能减少延迟。第三，同一个词在不同的地方，意思可能相同，也可能不同，那应该进行关联，比如我第二次选择翻译这个词，那应该要能基于当前的场景给予我合适的解释，也要能召回我之前的记忆，这样，符合人脑的网状记忆，学习效率更高。经过调研后，我发现 pdf.js 选词太不准确了，很影响阅读体验和学习效率。所以我决定采用以下架构：采用 SwiftUI (前端) + Rust (后端) + UniFFI (桥接) 的强类型单体架构。
